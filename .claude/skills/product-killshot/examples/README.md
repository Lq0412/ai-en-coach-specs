# product-killshot 示例用例

---

## 用例 1：全自动竞品分析

**用户输入：**

> 产品杀手锏，分析一下当前项目。

**期望 agent 行为要点：**

- 不追问用户，自动扫描 README、依赖清单、核心源码推断定位
- Web 搜索找到头部竞品，从竞品重度用户与本产品重度用户双视角分析
- 输出带优先级的 Markdown 优化报告并保存到文件

---

## 用例 2：指定竞品对比

**用户输入：**

> 竞品分析：我们的 CLI 工具 vs Vercel CLI，怎么打败他们？

**期望 agent 行为要点：**

- 仍先理解本项目代码与定位，但以用户指定竞品为主对比对象
- 报告含 switching cost、10x 改进点、功能缺口与路线图建议
- 结论可执行，避免空泛 SWOT

---

## 用例 3：功能规划导向

**用户输入：**

> 产品改进：接下来三个版本应该做什么功能？

**期望 agent 行为要点：**

- 基于代码现状 + 竞品能力矩阵，给出分版本功能规划
- 每项建议附带用户价值与实现复杂度粗估
- 标注 Quick wins vs 长期护城河
