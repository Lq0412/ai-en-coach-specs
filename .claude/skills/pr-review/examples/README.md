# PR Review — 示例

## 示例 1：Review 一个 PR

**用户：** 「帮我 review 一下 #56，是个 ASR WebSocket 集成」

**期望行为：** 按 4 个维度（正确性/安全性/架构/可维护性）逐条输出 Review 意见，区分 `[阻塞]` `[建议]` `[疑问]`。

## 示例 2：合入前检查

**用户：** 「PR #56 有 1 个 Approve 了，能合了吗」

**期望行为：** 逐条检查 4 个合入条件：Approved ✓ / 本地验证？/ rebase 了？/ 意见都处理了？→ 输出还缺什么。

## 示例 3：AI 代码审查

**用户：** 「这个 PR 的代码是 AI 生成的，帮我专项审查」

**期望行为：** 追加 code-discipline 的禁止项检查：兜底代码？猜测字段？冗余异常处理？→ 要求提交者逐段解释逻辑。
