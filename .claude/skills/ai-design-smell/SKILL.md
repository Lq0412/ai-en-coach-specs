# AI 设计味识别与修复

一眼识别 AI 生成 UI 的「丑设计味」，并给出可直接套用的修复方向。生成或评审界面前对照执行。

目标风格：**现代简约 —— 清晰层级 + 左对齐 + 留白足，单主色无渐变，拒绝模板化科技元素。**

## 五大识别特征（AI 丑设计一眼识别）

1. **卡片套娃太严重** —— 大卡套小卡，还堆图标按钮
2. **蓝紫渐变 + 发光玻璃按钮** —— 科技感虽有但超模板化
3. **文字大小没区别** —— 标题、正文、标签都一样大
4. **啥都居中对齐** —— 阅读动线完全没有
5. **标题上堆超大图标** —— 抢戏又拥挤

## 修复提示词模板（逐条对应）

1. 卡片用**简约单层**，少嵌套、留白多
2. 颜色选**单色系**别渐变，按钮**别发光**
3. 文字按 **标题 ＞ 副标题 ＞ 正文 ＞ 按钮** 区分大小
4. **主要左对齐**，分区布局留引导
5. 图标**别超标题高**，单个简约不堆叠

## 通用避坑（一句话）

现代简约风，清晰层级 + 左对齐 + 留白足，单主色无渐变，拒绝模板化科技元素就对啦。

## 用法

### 模式 A：评审（Review）

对现有 UI / 截图 / 前端代码逐条对照五大特征：

```
AI 设计味体检
─────────────
1. 卡片套娃      [✅ / ⚠️ 命中]  证据：<具体位置>  → 建议：<修复模板第1条>
2. 蓝紫渐变/发光  [✅ / ⚠️ 命中]  证据：...        → 建议：...
3. 文字层级缺失  [✅ / ⚠️ 命中]  证据：...        → 建议：...
4. 全部居中      [✅ / ⚠️ 命中]  证据：...        → 建议：...
5. 超大图标      [✅ / ⚠️ 命中]  证据：...        → 建议：...
─────────────
总评：命中 N/5 → <现代简约修复方向>
```

- 命中项给**具体证据**（哪个组件、哪段样式），不空泛
- 每条命中直接对应上面的修复模板

### 模式 B：生成 / 改写（Fix）

生成或重构界面时，把「修复提示词模板」作为硬约束注入：

- 卡片：单层、无多余嵌套、留白充足
- 颜色：单主色，无蓝紫渐变，按钮无发光/玻璃拟态
- 字号阶梯：标题 ＞ 副标题 ＞ 正文 ＞ 按钮，对比明确
- 对齐：正文与分区**左对齐**，仅必要处居中（如单一 CTA、空状态）
- 图标：尺寸 ≤ 同行标题高度，单个、简约，不成堆

## 自检清单（定稿前）

- [ ] 卡片最多一层嵌套，没有「卡里套卡再塞按钮」
- [ ] 没有蓝紫渐变、没有发光/玻璃按钮
- [ ] 标题/正文/标签字号肉眼可分（至少 3 级阶梯）
- [ ] 主内容左对齐，有清晰阅读动线
- [ ] 图标不高于标题、不堆叠
- [ ] 单一主色贯穿，留白充足

任一未过 → 按对应修复模板调整后再交付。

## 不适用

- 品牌方明确要求渐变/科技炫酷风（尊重需求，但可提示模板化风险）
- 数据可视化/图表本身的配色（另有规则）
- 非 UI 的纯文档排版
