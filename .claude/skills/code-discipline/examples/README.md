# code-discipline 示例用例

---

## 用例 1：审查冗余与猜测性代码（禁兜底）

**用户输入：**

> 看看这段代码有没有兜底、冗余或猜测性字段。

**期望 agent 行为要点：**

- 指出 silent fallback、过度防御、未确认就加的字段
- 给出最小改动重写，失败路径应明确报错
- 不清楚业务规则时主动追问，不自行假设

---

## 用例 2：复杂需求 — 先最小方案再确认

**用户输入：**

> 给订单列表加导出：筛选、分页、定时任务、邮件通知，全都要。

**期望 agent 行为要点：**

- 先给懒/最小方案（如当前页 CSV 一键下载），同条回复问「Y 是否够用？」
- 不擅自搭完整 pipeline；用户确认后再扩展
- 最多 3 行说明跳过了什么、何时再加

---

## 用例 3：决策阶梯 — 原生优先于新依赖

**用户输入：**

> 表单里要加个日期选择，帮我装个 date picker 库。

**期望 agent 行为要点：**

- 按决策阶梯：先问是否真需要复杂 picker；简单场景用 `<input type="date">` 或平台原生能力
- 不擅自 `npm install` 新依赖；若跳过库，最多 3 行说明何时再加（时区、范围限制、自定义样式等）
- 代码优先给出最小可用实现
