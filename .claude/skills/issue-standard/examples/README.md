# Issue Standard — 示例

## 示例 1：从调研结论创建 Issue

**用户：** 「市场调研确定了 ASR 用 Deepgram，帮我建对应的 Issue」

**期望行为：** 输出符合模板的 Issue：`[功能] ASR Deepgram 流式语音识别集成`，正文包含背景（引用调研结论）、要做的事（封装音频模块/WebSocket/VAD）、验收标准（可验证的 4 条）、关联（依赖的 MS 和文档链接）。

## 示例 2：拆分过大 Issue

**用户：** 「这个 Issue 写的是"ASR 集成 + 发音评测 UI + 课后总结页"，怎么拆」

**期望行为：** 按拆分规则（跨技术域 / 超 1 人天）拆成 3 个独立 Issue，各自可独立开发和验收。

## 示例 3：标签补全

**用户：** 「这个 Issue 只挂了 P0，帮我看看还缺什么标签」

**期望行为：** 检查 Issue 内容 → 输出推荐标签（如 `后端` `调研`）。
